# Службы времени

## Ручная настройка

### Настройка системных часов

![[Pasted image 20250424154247.png]]

Задать время вручную (привилегированный режим)

```cisco
clock set hh:mm:ss {day month | month day} year
```

Минус:

- Нужно поставить точно
- При перезагрузке, время считается с момента начала перезагрузки (время слетает)

Ручная установка - неудачный способ настройки

Лучше использовать NTP

### Пример ручной настройки часов

```cisco
clock set 19:30:00 may 13 2023
```

## NTP

### Работа протокола NTP

![[Pasted image 20250427151036.png]]

Работают иерархически. Доверие к Stratum 0 - должны быть максимально точными

Максимальное количество переходов 15, UDP, 123 порт, [rfc1305](https://datatracker.ietf.org/doc/html/rfc1305)

>stratum (англ.)- слой

Серверы на одном уровне могут также синхронизироваться между собой

### Настройка NTP (клиент + сервер)

![[Pasted image 20250427151357.png]]

Текущее время

```cisco
show clock
```

Настройка клиента

```cisco
ntp server ip-address
```

ip-address - источник информации о времени

- Настраиваемый маршрутизатор должен иметь stratum на один больше, чем тот, клиентом которого является маршрутизатор

Настройка сервера (где указывает stratum)

```
int <int_name>
ntp broadcast
ntp master [stratum]
```

Можно держать локальный сервер ntp и синхронизироваться с ним

![[Pasted image 20250427152032.png]]

- address - ip-адрес NTP-сервера
- ref (устройство, которое выступает сервером для устройства, у которого запрашивается время)

То есть маршрутизатор в этом случае находится на 2 стратуме
